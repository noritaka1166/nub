# Each non-comment line is a substring matched against a failing jest test's
# full name ("<describe> > <it>" as jest reports it, or a bare test-file path).
# A failure whose name matches ANY line here is reported as KNOWN-FAILING (not a
# surprise) and does not fail the run. A failure matching NO line is a SURPRISE
# and fails the run. Keep this list SHORT and each entry justified — a stale
# entry (a test that now passes but is still allowlisted) is also flagged, so
# this file shrinks as bugs are fixed and stays honest about intended divergence.
#
# Format:  one match-substring per line; blank lines and #-comments ignored.
# Comment the WHY directly above each cluster.

# ── Codified INTENTIONAL divergences (nub is deliberately not pnpm here) ──────
# These are product decisions, documented in the wiki — nub diverges on purpose.

# (1) `nub upgrade` is self-update of the nub binary, NOT pnpm's `update` alias.
#     pnpm aliases `pnpm upgrade` -> `pnpm update`; nub does not. Any test that
#     drives `pnpm upgrade` expecting dependency-update behavior diverges.
upgrade

# (2) No implicit script shortcuts. pnpm lets `pnpm <script>` run a package.json
#     script without `run`; nub requires the explicit `run` verb. Tests that rely
#     on the bare-script shortcut diverge.
should run a script that is specified as a command shortcut
runs a script
command shortcut

# (3) Top-level file-run command surface. `nub <file>` runs a file; pnpm has no
#     such top-level verb. Any suite test depending on pnpm's top-level argument
#     handling for non-verb tokens may diverge.
# (no current black-box test exercises this directly; placeholder cluster kept
#  empty intentionally — add a precise match only if a real test surfaces it.)

# (4) Global root layout. `pnpm root -g` prints
#     <pnpm-home>/global/<LAYOUT_VERSION>/node_modules; nub uses its own global
#     layout (<pnpm-home>/global-nub). Surfaced by this harness on 2026-06-20
#     (test/root.ts "pnpm root -g"). Whether to mirror pnpm's exact path is a
#     product decision (global install layout) — allowlisted as a known
#     divergence pending that call, NOT auto-changed.
pnpm root -g
