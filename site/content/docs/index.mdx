Nub is an all-in-one toolkit for Node.js, written in Rust. One binary runs your TypeScript files and scripts, installs your dependencies, and provisions Node itself — replacing `node`, `npm`/`pnpm`, `npx`, and `nvm` without adopting a new runtime.

```bash
npm install -g --ignore-scripts=false @nubjs/nub
nub index.ts             # run a TypeScript file
nub install              # install dependencies (pnpm-compatible)
nub run dev              # run a package.json script
nubx prisma generate     # run a CLI from node_modules/.bin
```

There is no new runtime to adopt and no lock-in: every augmentation rides on Node's own public extension surfaces, and the package manager reads and writes whatever lockfile your project already uses.

## The toolkit

- **A runtime** — execute `.{js,cjs,mjs,jsx,ts,cts,mts,tsx}` files with full TypeScript support, on the stock `node` binary. See [files](/docs/runtime).
- **A package manager** — a pnpm-compatible installer that round-trips your existing npm, pnpm, or bun lockfile. See [install](/docs/install).
- **A script runner** — run `package.json` scripts 24× faster than `pnpm run`, with the full workspace and lifecycle-hook surface preserved. See [run](/docs/run).
- **A bin runner** — run a local CLI an order of magnitude faster than `npx`, or fetch and run an uninstalled one from the registry. See [nubx](/docs/nubx).
- **A Node version manager** — pin a version and Nub provisions it from nodejs.org on demand, replacing `nvm` and `fnm`. See [node](/docs/node).
- **A watcher** — restart on changes to your source, your `.env*` files, and your `tsconfig.json`, driven by the dependency graph. See [watch](/docs/watch).
- **A GitHub Action** — swap `actions/setup-node` for `nubjs/setup-nub` to install Nub, warm the project's Node, and cache Nub's store in CI. See [GitHub Action](/docs/setup-nub).

## What Nub replaces

| Nub | What it does | Replaces |
|---|---|---|
| `nub <file>` | Runs `.js`/`.ts`/`.jsx`/`.tsx` on stock Node, transpiling TypeScript and JSX, loading `.env*` files, and polyfilling modern globals per Node-version band. A drop-in for `node <file>`. | `node`, `tsx`, `ts-node`, `dotenv-cli`, `node --env-file`, `deno run`, `bun run` (for the TS/JSX/`.env` cases) |
| `nub run <script>` | Runs a `package.json` script with the full lifecycle-hook and workspace surface, including recursive runs with `--filter` and topological ordering. Not a task-graph runner with cross-run caching. | `npm run`, `pnpm run`, `yarn run`, `bun run`, `pnpm -r` (recursive scripts) |
| `nubx` / `nub dlx` | Runs a binary from `node_modules/.bin`, fetching it from the registry on a local miss. | `npx`, `pnpm dlx`, `pnpm exec`, `yarn dlx`, `bunx` |
| `nub install` (and the full PM surface) | The complete package-manager surface: install, add/remove/update, dedupe, audit, pack/publish, and more. pnpm-compatible CLI; round-trips npm, pnpm, and bun lockfiles; reads yarn. | `npm`, `pnpm`, `yarn`, `bun` (as a package manager) |
| `nub watch` | Restarts the entry point on changes to its resolved import graph, `.env*` files, `tsconfig.json`, and manifest. Restart-mode, on Node's own `--watch`; not HMR. | `nodemon`, `node --watch`, `tsx watch`, `bun --watch` |
| `nub node` | Provisions and pins Node versions, fetching from nodejs.org on demand (`install`, `ls`, `uninstall`, `pin`). Also provisions the exact package-manager version a project pins in `packageManager`. | `nvm`, `fnm`, `n`, `volta`, `asdf` (the Node plugin), `corepack` |

## Install

```bash
npm install -g --ignore-scripts=false @nubjs/nub
```

Both `pnpm add -g @nubjs/nub` and `yarn global add @nubjs/nub` work equivalently. That installs the Rust binary plus the platform-specific N-API addons, and puts `nub` and `nubx` on your `PATH`. In CI, swap `actions/setup-node` for the [setup-nub action](/docs/setup-nub):

```diff
- - uses: actions/setup-node@v4
+ - uses: nubjs/setup-nub@v0
```

## Package management

The installer is pnpm-compatible at the CLI surface and lockfile-compatible with whatever your project already uses. It infers the incumbent package manager and mirrors it — npm, pnpm, and bun lockfiles round-trip cleanly; yarn is read-only.

```bash
nub install              # resolve + link node_modules into the existing lockfile
nub add -E -D react      # exact pin, devDeps — pnpm flag spelling
nub remove lodash
nub update
nub ci                   # frozen-lockfile install
```

It's optional: keep running your existing package manager and reach for Nub for everything else. See [install](/docs/install) for lockfile fidelity per incumbent, and [pm](/docs/pm) for provisioning the exact package-manager version a project pins.

## Requirements

- **Node 18.19+** (Node 18 LTS) for augmented modes.
- macOS (arm64, x64), Linux (x64, arm64), Windows (x64, arm64).
